KEYBOARD SHORTCUTS


It is very common for extensions to have keyboard conflicts.
We always provide a way to use either a menu or a keyboard shortcut in these situations,
so if you have a problem opening something with a shortcut, try the menu option next.

You can report the keyboard conflicts to me and I will find a way to help you.
It is very common for third party keyboards to have different control keys on the Macintosh platform.

The known list of Applications and Extensions who have Keyboard Shortcuts that can conflict with FastTrack

CHECK THE ERROR CONSOLE


Close all other Windows and Tabs except this one.

Select Tools -> Error Console
Click on ALL, then Click on Clear

Select Tools ->Addons -> gj53run -> Options
Click on Debug Extension Messages
Click on Debug Annimation Messages
Close the Addons window


Go here and make sure alternate debugging is set on

Open the problem page in a new Tab and repeat the problem action.

View the messages in the console.
Its common to see many errors and messages in debug mode.
You should probably contact me and set up a phone call or a chat to interpert these

Clear the console after each viewing to make it easier to isolate the errors you are getting.

Don't forget to turn off the debugging options when your done!

SOUND


1) The very first thing to do is check your volume controls to make sure the volume is on (not muted)
and that it is set high enough for you to hear it.

2) Click on the button "Run Checks" above

Scroll down this page and find the line "Wav Plugins", if the line is blank after wav plugins, you need to install a plugin for wav files

If you do have a wav plugin installed, find the line "Your Install Directory"
From your Operating System's file Explorer,  go to your Install Directory -> XXSoftwareTools -> Sounds
click on a wav file to see what by default plays wav files for your OS.
If you do not hear anything, check your volume controls or contact an Operating System support person
