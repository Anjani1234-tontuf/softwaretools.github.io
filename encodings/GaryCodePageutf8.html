You can refer to these applications as multi-byte aware.

A utf (multi byte, without a BOM) file that contains:
   no multi byte characters in the first 1000 bytes
   and is mostly code points below 128
   with a few multi byte characters, where the first of the multi bytes are C2 or C3
   (the bytes we show in the images above)
   and do not contain a code page directive like HTML or xml files
   (example <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">)
   are the most error prone.

Searching for U+00F6 using Perl regular expressions
The code point is U+00F6, what you see in a hex editor is 0xc3 0xb6

* Thanks to Ben at IDM Computer Solutions (Ultraedit, UE Studio) for making me aware of this
"There is a difference between a Unicode character's code point and hex value.

The code point of a Unicode character is the same across all Unicode formats (UTF-8, UTF-16, UTF-32, etc.).
\xF6 will match the same character across all formats.

The hex value of a Unicode character varies from encoding to encoding.
"C3 B6" is the unique hex value of UTF-8's encoding of the ö character."

When you search with Perl regex via "\x", you're searching for its code point, not its hex value.
Perl was deliberately designed this way.

Examing the bits of Utf-8 codes

In utf-8, bytes with a numeric value greater than 127 are part of a multibyte sequence.

    110xxxxx One more byte follows
    1110xxxx Two more bytes follow
    11110xxx Three more bytes follow

Finally, the bytes that follow those start codes all look like this:

    ASCII characters of one byte always start with a 0 bit.
    The first byte of a size two character always starts with bits 110.
    The first byte of a size three characters always starts with the bits 1110.
    The first byte of a size four characters always starts with the bits 11110.
    A follow up byte of a multi byte character always starts with 10.
    This makes it impossible for a follow up character to look like a carriage return or a quote.

The octet values C0, C1, F5 to FF never appear.
